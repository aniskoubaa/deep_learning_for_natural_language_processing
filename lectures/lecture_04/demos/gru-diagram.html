GRU: Gated Recurrent Unit

A simplified gating mechanism with only two gates (reset & update) and no separate cell state — fewer parameters, similar performance to LSTM on many tasks.

Cell body
Sigmoid gate (σ)
Tanh layer
Pointwise op
Vector flow
Hidden state
1. Reset Gate
$$r_t = \sigma\big(W_r \cdot [h_{t-1},\, x_t] + b_r\big)$$
Decides how much past memory to forget when computing the candidate. When r_t ≈ 0, the candidate ignores h_{t-1} entirely — like starting fresh.
2. Update Gate
$$z_t = \sigma\big(W_z \cdot [h_{t-1},\, x_t] + b_z\big)$$
Controls the balance between old and new. Acts as both LSTM's forget gate and input gate combined: z_t ≈ 1 means "keep old h", z_t ≈ 0 means "use new candidate".
3. Candidate + Output
$$\tilde{h}_t = \tanh\big(W \cdot [r_t \odot h_{t-1},\, x_t] + b\big)$$ $$h_t = (1 - z_t) \odot h_{t-1} \;+\; z_t \odot \tilde{h}_t$$
The candidate uses the reset-filtered past. Then the update gate interpolates between old h and new candidate — a smooth blend.
